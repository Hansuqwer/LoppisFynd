# Onboarding Gate Screen - Nature Distilled

## Purpose

- Decide whether to show onboarding or go straight to the app.

## UX Rules

- First run: show `roadmapv2/research/onboarding_screen.md`.
- Returning users: skip onboarding entirely.
- If we add new onboarding steps later: show a "What’s new" mini-step only (1-2 cards), not the full onboarding.

## Layout

Use the same gate layout as auth gate for consistency.

## Tokens (Color + Typography)

Use v2 ramps from `roadmapv2/research/color_palette_tokens.md` and text styles from `roadmapv2/research/typography_tokens.md`.

Color:
- background: neutral0
- progress accent: terracotta500

Typography:
- status: bodySmall
