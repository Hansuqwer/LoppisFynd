---
name: brainstorm
description: Explore options before implementation. Use when the user says /brainstorm or asks to compare approaches, risks, product options, architecture options, or implementation strategies.
---

Stay read-only unless the user explicitly asks for edits.

Workflow:

1. Restate the goal and the decision that needs to be made.
2. List 3-5 viable options with tradeoffs.
3. Recommend one default path and explain why.
4. Identify the smallest next step that would validate the choice.

Do not write code, change files, run migrations, or install dependencies in this workflow.
