<objective>
Make sold-price comps a reliable, user-visible feature with a true OFF switch: when enabled, users can refresh comps (on demand or via background sync) and see last-updated time; when disabled, the app performs zero comps network calls and the UI clearly indicates why refresh is unavailable.

Purpose: Close Phase 3 reliability gaps where disabling comps still allows background/manual churn and where the UI hides the “last updated” signal.
Output: Hardened gating across all market-sync entrypoints + item detail UI that shows last updated and disables comps actions when off.
</objective>

  <action>
Make the comps enable/disable toggle authoritative for ALL comps sync entrypoints (manual + background) so disabling comps guarantees no comps network calls and no churny status/error updates.

Rules (must match existing privacy-toggle behavior):
- Use `kPrivacyFetchSoldPriceCompsEnabledKeyV1` (int) from `lib/core/settings/app_settings_keys.dart`.
- Treat missing/unset as enabled (`1`).
- Treat any non-`1` value as disabled.

Required changes:
- `lib/services/sync/background/background_sync.dart`
  - In `BackgroundSync.scheduleIfConfigured(...)`, before reading the interval and scheduling work, read the comps enabled key from `db.appSettingsDao`.
  - If comps are disabled, call `Workmanager().cancelByUniqueName(_taskName)` and return.
  - In `callbackDispatcher()` task execution, open the DB and read the comps enabled key before constructing `TraderaClient` / `MarketBridge`.
  - If disabled, return `true` immediately (no network, no status updates).

- `lib/services/sync/sync_scheduler.dart`
  - At the beginning of `_syncOnceInternal()`, read the comps enabled key from `_db.appSettingsDao`.
  - If disabled, return early BEFORE:
    - reading/incrementing quotas
    - listing pending items
    - transitioning any item status
    - calling `_market.fetchComps(...)`
  - Keep `SyncRunStarted` / `SyncRunFinished` event semantics intact (no unhandled exceptions).

- `test/services_sync/fl_042_sync_scheduler_test.dart`
  - Add a regression test proving that when comps are disabled:
    - `SyncScheduler.syncOnce()` does not call the market data source (use a throwing `MarketDataSource`).
    - quota is not incremented.
    - no sync state error is recorded.
    - item status is not transitioned.
  </action>
  <verify>
1) `flutter analyze`
2) `flutter test test/services_sync/fl_042_sync_scheduler_test.dart`
  </verify>
  <done>
- With comps disabled, Workmanager does not schedule/run `market_sync` and `SyncScheduler` performs no network calls or status churn.
- Tests enforce the no-network-call contract when disabled.
  </done>
</task>

Make the Item Detail market section clearly communicate comps state and provide a "last updated" signal.

UI behavior requirements (Phase 3 success criteria):
- If comps are disabled (Settings toggle OFF):
  - Disable the comps actions (`Queue sync` / `Sync now`) by setting `onPressed: null`.
  - Show a short hint that comps are disabled + provide an "Open settings" button.
- If comps are enabled but the app is offline (or connectivity unknown):
  - Disable comps actions and show an offline hint (do not attempt calls).
- If comps are enabled but `TRADERA_PROXY_URL` is not configured (`!config.hasTraderaProxy`):
  - Disable comps actions and show a proxy-not-configured hint.
- When comps data exists for the item, show a visible "Last updated" timestamp sourced from `ScanItemComps.fetchedAt`.
- If the last sync attempt failed for this item, surface a concise error state (use `ScanItemSyncStates.lastError` and/or `nextAttemptAt`).

Implementation guidance:
- In `ItemDetailScreen.build(...)`, watch `fetchSoldPriceCompsEnabledProvider`, `isOnlineProvider`, and `appConfigProvider`.
- Add `watchByScanItemId(...)` to `ScanItemSyncStatesDao` so the UI can reactively show `lastError` / `nextAttemptAt`.
- Prefer existing design language used for cloud identify disabled hint + "Open settings" (reuse `GlassButton` patterns already on the screen).
- Keep core flows intact: only gate comps-specific actions; do not break item editing or listing/drafting actions.
  </action>
