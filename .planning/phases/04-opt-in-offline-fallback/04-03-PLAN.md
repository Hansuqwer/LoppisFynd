  <action>
Implement the Phase 4 evidence UI as a dedicated screen to keep complexity out of ItemDetail and Scanner.

- Create `OfflineDetectionScreen` that can be opened with a `scanItemId`.
- UI requirements (locked decisions):
  - Split view shows the image with bounding boxes AND a results list together (responsive layout allowed as long as both are visible).
  - Tapping a box selects the corresponding list row; selecting a list row highlights the box; keep selection in sync.
  - Show confidence as percent plus High/Med/Low label; do not show detections below the hard threshold.
  - If multiple objects are detected: show multiple boxes and let user pick the primary (selected).
- Model-missing UX (locked decisions):
  - When triggered and model is not installed, show a non-blocking inline card in the detection panel with a Download button.
  - Download UI must show size up front, progress bar, and Pause/Cancel controls.
  - Pause must preserve partial bytes for resume; Cancel must delete partial.
  - On failure, show an error but allow user to continue (detection remains unavailable).
- Best-effort auto-run (locked decisions):
  - When the screen opens, automatically start detection if offline mode is enabled and model is installed.
  - Do not block navigation/other UI while detection runs; show an in-panel loading state.
- Prefill-only side effects (locked decisions):
  - Provide an explicit action to apply the selected detection to fields (e.g., set query/category suggestions). Do not auto-save changes until user taps an action.
  </action>

  <done>
OfflineDetectionScreen renders image+boxes and list together, selection sync works both ways, low-confidence detections are hidden, and the inline download card supports download/pause/resume/cancel with visible progress.
  </done>
