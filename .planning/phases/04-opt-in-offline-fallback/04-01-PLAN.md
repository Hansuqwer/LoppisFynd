  <action>
Update `ModelManager.downloadFromUrl` so it can be used for the Phase 4 inline download card with progress + pause/cancel.

- Add a new optional callback parameter: `bool Function()? isPaused`.
- Semantics:
  - Pause: stop transferring, KEEP the `.partial` file, and leave any previously installed model untouched.
  - Cancel: stop transferring, DELETE the `.partial` file, and leave any previously installed model untouched.
- Implementation approach:
  - Introduce a distinct exception (e.g. `_ModelDownloadPausedException`) and throw it when `isPaused?.call() == true`.
  - Ensure the `catch` cleanup block deletes `.partial` for errors/cancel, but NOT for pause.
  - Keep current cancellation behavior for `isCancelled` (existing tests rely on cleanup).
- Update/add tests in `test/services_ai/fl_030_model_manager_test.dart`:
  - Add a new test that pauses mid-download and asserts `.partial` still exists and the installed file remains unchanged.
  - Add a resume call that continues via Range request (assert `Range: bytes=<existing>-`) and completes successfully.
  - Keep existing tests passing (progress, resume-from-partial, cleanup-on-error, cleanup-on-cancel).
  </action>
  <verify>flutter test test/services_ai/fl_030_model_manager_test.dart</verify>
  <done>
Pause preserves partial bytes for resume; Cancel/error cleanup behavior remains as before; all ModelManager tests pass.
