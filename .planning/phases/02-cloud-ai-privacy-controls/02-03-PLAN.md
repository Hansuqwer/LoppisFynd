Implement the cloud identification backend in the Flutter app while enforcing PRIV-03:

Config:
- Add `CLOUD_AI_PROXY_URL` to `AppConfig.fromEnvironment()` with default empty string and `hasCloudAiProxy` helper.

Client contract:
- Create `lib/services/ai/cloud_ai_proxy_client.dart` that posts to `CLOUD_AI_PROXY_URL`.
- Request body must include `prompt`, `maxTokens`, and `imageBase64Jpeg` (crop only).

Crops-only + metadata stripping:
- Implement `lib/services/ai/image_cropper.dart` using `package:image` that:
  - Decodes the original photo, creates a crop (start with a center square crop), downsizes to a max dimension (e.g. 768px), re-encodes to JPEG.
  - Re-encoding must strip metadata by construction (no EXIF passthrough).
  - Enforces a byte-size budget; if exceeded, downscale further or fail with a clear error.
- Ensure the original full photo bytes are never sent over the network.

AI abstraction:
- Extend `AiBackendKind` in `lib/services/ai/inference/inference_isolate_service.dart` to include `cloudGemini`.
- Wire `AiInferenceIsolateService.run(...)` to call the proxy when `cloudGemini` is selected.
- Preserve cancellation behavior (if cancellation happens during a request, stop processing and return `AiCancelledException`).

Default backend:
- Update `lib/main.dart` so the injected `AiInferenceIsolateService` uses `AiBackendKind.cloudGemini` by default when `CLOUD_AI_PROXY_URL` is configured.

Privacy gating integration:
- Reuse the gating logic from Plan 02-02: only allow cloud call when user consent is accepted and cloud toggle ON and online.
