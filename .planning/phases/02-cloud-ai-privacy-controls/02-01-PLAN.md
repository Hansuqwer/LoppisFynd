  <action>
Create a new Supabase Edge Function in `supabase/functions/cloud-ai-proxy/` that:

- Exposes `POST` (and `OPTIONS` for CORS) and returns JSON.
- Reads secrets from `Deno.env`: `GEMINI_API_KEY` (required) and `GEMINI_MODEL` (optional; default to a reasonable Gemini model).
- Accepts a JSON body shaped like:
  - `prompt: string` (required)
  - `maxTokens?: number`
  - `imageBase64Jpeg: string` (required)  # base64 bytes only (no data: prefix)
  - `requestId?: string` (optional, for logging-safe correlation)
- Validates:
  - Reject missing/empty prompt or image with 400.
  - Reject obviously huge payloads with 413 (size guard; base64 length budget).
- Calls Gemini upstream server-side using `fetch` and the API key (do not log request body; do not persist bytes).
- Returns `{ text: string, raw: object }` (or `{ error: string }` on failure). Keep raw truncated if needed.
- Sets response headers to discourage caching: `cache-control: no-store`.
- Ensure error messages are actionable but do not include the base64 payload.

Follow the established CORS + JSON style used by `supabase/functions/tradera-proxy/index.ts`.
  </action>
