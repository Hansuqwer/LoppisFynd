  <name>Task 1: Gate scan-capture background inference on disclosure + toggle + online + proxy</name>
  <files>lib/features/scanner/scan_capture_service.dart</files>
  <action>
Update scan-capture background inference so it cannot call cloud AI unless the user explicitly allowed it.

Implementation requirements (match Phase 2 locked decisions):
- Before calling `_aiInference.run(...)`, check BOTH:
  - `kPrivacyCloudIdentificationEnabledKeyV1 == 1` (default ON when unset)
  - `kCloudIdentificationDisclosureChoiceKeyV1 == 1` (accepted; treat 0/2 as disallowed)
- Also gate on environment/runtime readiness:
  - `AppConfig.hasCloudAiProxy == true` (no proxy configured => do not attempt cloud)
  - Online check (fail closed to "skip"; do not queue). Implement this as an injected async callback on the service constructor:
    - Add `Future<bool> Function()? isOnline` to `ScanCaptureService(...)`
    - Default implementation uses `connectivity_plus` to check connectivity
    - Wrap the default in try/catch; if connectivity cannot be determined, treat as online to avoid breaking capture.

Wiring details:
- Keep thumbnail generation and DB writes unchanged and always executed.
- Place the gating immediately before the current inference block (right before computing `modeKey` / `maxTokens`).
- If gating fails, return early from the inference block (do not transition status to failed; keep existing status).

Notes:
- This task intentionally enforces "no upload" by skipping the entire background inference path when disallowed, even if a local backend might exist.
  </action>
