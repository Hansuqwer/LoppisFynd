Replace the current onboarding Gemma consent UX with a Phase 2 cloud identification disclosure:

- On first app launch (onboarding), show a blocking full-screen/modal disclosure with exactly two choices:
  - "Enable cloud identification"
  - "Not now"
- The disclosure copy is short (2-4 bullets) and includes a "Learn more" link.
- "Learn more" explains:
  - Provider is described generically as "cloud AI" (do not name Gemini/provider)
  - Upload is crops only, never full photo
  - Metadata is stripped
  - Uploaded image data is processed transiently and not stored on our server
  - Optional preview of what would be uploaded (crop preview) if an image is available (OK to show a placeholder when no image exists during onboarding).

Persisted state:
- Store the disclosure choice in `AppSettingsDao` as an int (0 unknown / 1 accepted / 2 not now).
- The choice must be reversible via Settings (turning the cloud toggle OFF should disable cloud identification regardless of prior acceptance).

Just-in-time behavior in Identify:
- In `lib/features/analyzer/item_detail_screen.dart`, gate Identify so that:
  - If cloud toggle is OFF: Identify appears disabled with a short hint and an "Open Settings" shortcut.
  - If user is offline while cloud toggle is ON: tapping Identify fails fast with "You're offline" and Retry + "Open Settings" shortcuts; do not queue.
  - If disclosure choice is "Not now" (or unknown): tapping Identify re-prompts with the same disclosure and requires Enable to proceed.

Do not implement the cloud call in this plan; this plan only builds the disclosure + gating behavior and persistence.
